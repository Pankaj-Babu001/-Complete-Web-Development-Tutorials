// IMPERATIVE: You tell HOW to do it (step-by-step) let count = 0; function updateDisplay() { const el = document.getElementById('counter'); el.textContent = count; // Manually update DOM if (count < 0) { el.style.color = 'red'; // Manually update styles } else { el.style.color = 'blue'; } }
// DECLARATIVE: You tell WHAT you want (describe the outcome) function Counter() { const [count, setCount] = useState(0); return ( <div style={{color: count < 0 ? 'red' : 'blue'}}> {count} </div> ); // React figures out HOW to update the DOM }
  • Imperative: You manually select elements, update their content, modify styles, and manage every detail of how the DOM changes.
  • Declarative: You describe what the UI should look like for any given state, and React automatically figures out the most efficient way to update the DOM.
  • Mental Model: Imperative is like giving turn-by-turn directions; Declarative is like setting a destination in GPS.
  • Maintenance: Declarative code is easier to reason about because the UI is a direct reflection of state, not scattered DOM manipulations.
  • React's Magic: React uses a Virtual DOM to calculate the minimal set of changes needed, making declarative code performant.
