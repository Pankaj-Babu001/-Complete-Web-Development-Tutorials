Concept: Abstract the repetitive code into reusable functions

// Custom React Object
const CustomReact = {
  createElement: function(tag, attributes, children) {
    const element = document.createElement(tag);
    element.textContent = children;

    for(const key in attributes) {
      if(key === 'style') {
        Object.assign(element.style, attributes.style);
      } else {
        element[key] = attributes[key];
      }
    }

    return element;
  }
};

// Custom ReactDOM
const CustomReactDOM = {
  render: function(child, parent) {
    parent.append(child);
  }
};

3Ô∏è‚É£ Approach 3: Virtual DOM Concept (The React Way)

Concept: Create JavaScript objects first, then convert to DOM

üìù The Code:

// Virtual React - Returns Objects, not DOM elements
const VirtualReact = {
  createElement: function(type, props, children) {
    return {
      type: type,
      props: {
        ...props,
        children: children
      }
    };
  }
};

// Virtual ReactDOM - Converts objects to real DOM
const VirtualReactDOM = {
  render: function(reactElement, root) {
    root.innerHTML = '';

    const element = document.createElement(reactElement.type);
    const {props} = reactElement;

    for(const key in props) {
      if(key === 'style') {
        Object.assign(element.style, props.style);
      } else if(key === 'children') {
        element.textContent = props[key];
      } else {
        element[key] = props[key];
      }
    }

    root.append(element);
  }
};

‚úÖ Pros:

  • Creates lightweight JavaScript objects first
  • Can compare objects before updating DOM
  • Foundation for optimization
  • Separation of concerns

üéì Key Concepts Summary

1. Real DOM vs Virtual DOM

Real DOM:
- Actual browser elements
- Heavy to create/update
- Direct manipulation

Virtual DOM:
- JavaScript objects
- Lightweight
- Indirect manipulation (through React)

2. The React Flow

1. Write Code ‚Üí React.createElement()
2. Create Virtual DOM ‚Üí JavaScript Object
3. Render ‚Üí ReactDOM.render()
4. Reconciliation ‚Üí Find differences
5. Update ‚Üí Minimal DOM changes
6. Display ‚Üí User sees result

3. Important Terms

  • Element: Object describing what you want to see
  • Component: Reusable piece of UI
  • Props: Data passed to elements/components
  • Children: Content inside an element
  • Reconciliation: Process of comparing and updating
